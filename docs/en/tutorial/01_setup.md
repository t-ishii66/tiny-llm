# Step 1: Setup and Run

Let's start by running the code.

---

## 1.1 Requirements

- Python 3.8 or higher
- PyTorch (CPU version is sufficient)

```bash
pip install torch
```

No GPU required. tiny-LLM completes training in seconds.

---

## 1.2 Getting the Code

```bash
git clone https://github.com/t-ishii66/tiny-llm.git
cd tiny-llm
```


## 1.4 Reading the Output

### Training Progress

```
epoch   20  loss=1.9469    ← Still mostly random predictions
epoch  200  loss=0.1147    ← Nearly perfect predictions
```

- **epoch**: How many times the model has gone through the training data
- **loss**: How bad the predictions are (lower is better). Random ≈ 2.3, perfect = 0

### Generation Results

```
prompt: "the cat sat on"
output: the cat sat on the mat . the dog sat on the log ...
```

- **prompt**: The input text given to the model
- **output**: Text generated by the model, predicting one word at a time

The model generates natural English sentences consistent with the training corpus.

> **Note**: The prompt passed to `generate()` must consist only of words that appear in the training corpus.
> Including out-of-vocabulary words will cause an exception in the current implementation.

---

## 1.5 What Just Happened

In just a few seconds, the following was executed:

1. **Data preparation**: Converted a 40-word corpus into numerical form and created training data
2. **Model construction**: Initialized a Transformer with ~68,000 parameters
3. **Training**: Learned to predict "the next word" over 200 training loops
4. **Generation**: Generated text using the trained model

In the next steps, we'll examine the internals of each stage with our own eyes.
